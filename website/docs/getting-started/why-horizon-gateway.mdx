# Why Horizon Gateway?

> **Use when:** your system already speaks NATS internally and you want one battle-tested, language-agnostic edge for HTTP clients without paying the per-service ingress cost.
> **You pick:** Horizon Gateway when you want type-safe route declaration in the SDK consumer, contract-enforced HTTP↔NATS translation, and a single point to terminate auth and rate limits.
